<Callout type="info">

</Callout>

## File Structure

The MDX blog uses a simple file-based structure:

```
src/app/[locale]/(public)/blog/mdx/
├── _files/
│   ├── introduction-next-js.mdx
│   ├── tailwind-css-v4.mdx
│   ├── guide-react-19.mdc
│   └── sample.mdx
├── [slug]/
│   └── page.tsx
└── page.tsx
```

**Key Locations:**
- **Content Files**: `_files/` directory contains all MDX content
- **Routing**: Dynamic `[slug]` route renders individual posts
- **Index**: `page.tsx` displays list of all MDX posts

## Creating MDX Content

<Steps>
<Step title="Create MDX File">
  Create a new `.mdx` or `.mdc` file in the `_files/` directory:

  ```bash
  # Navigate to the MDX files directory
  cd src/app/[locale]/(public)/blog/mdx/_files/

  # Create a new blog post
  touch my-awesome-post.mdx
  ```
</Step>

<Step title="Add Frontmatter">
  Start your MDX file with YAML frontmatter at the top of the file.

  **Frontmatter Fields:**
  - `title` - Post title (required)
  - `description` - Meta description for SEO
  - `publishedAt` - Publication date
  - `readTime` - Estimated reading time
  - `category` - Content category
  - `tags` - Array of tags
  - `author` - Content author
</Step>

<Step title="Write Content">
  Add your content using Markdown and React components. You can use:

  - **Headings** with # ## ###
  - **Callout components** for important information
  - **Tabs and Tab components** for organized content
  - **Code blocks** with syntax highlighting
  - **Interactive buttons** and other React components
  - **Lists, tables, and standard markdown**
</Step>
</Steps>

## Content Management

### File Naming

Use descriptive, URL-friendly filenames:

```bash
# Good examples
introduction-to-react.mdx
nextjs-performance-tips.mdx
typescript-best-practices.mdx

# Avoid
My Post!.mdx
post-1.mdx
untitled.mdx
```

### Content Organization

Organize content logically:

<Steps>
<Step title="Categories">
  Use consistent categories in frontmatter:
  - `"Development"` - Technical content
  - `"Tutorial"` - Step-by-step guides
  - `"News"` - Updates and announcements
  - `"Tips"` - Quick tips and tricks
</Step>

<Step title="Tags">
  Use specific, searchable tags like:
  `["nextjs", "react", "typescript", "performance"]`
</Step>

<Step title="Publishing Dates">
  Use consistent date format: `"2024-01-15"` (YYYY-MM-DD format)
</Step>
</Steps>

## Content Migration

### Export from Database Blog

Convert database posts to MDX:

```typescript
// Export database post to MDX
export async function exportToMDX(postId: string) {
  const post = await getPostById(postId)

  const frontmatter = {
    title: post.title,
    description: post.description,
    publishedAt: post.publishedAt.toISOString().split('T')[0],
    category: post.category?.name,
    tags: post.hashtags.map(h => h.name)
  }

  const mdxContent = `---
${yaml.stringify(frontmatter)}---

${post.content}
`

  fs.writeFileSync(`_files/${post.slug}.mdx`, mdxContent)
}
```
