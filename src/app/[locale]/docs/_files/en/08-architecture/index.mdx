The boilerplate implements a **strict 3-layer architecture** that ensures separation of concerns, maintainability, and scalability. Each layer has specific responsibilities and communicates through well-defined interfaces.

<Callout type="success">
  **Perfect for:** Building maintainable, testable, and scalable SaaS applications with clear separation between UI, business logic, and data access.
</Callout>

## Architecture Overview

The system follows a **unidirectional data flow** through three main layers:

<Tabs defaultValue="architecture-diagram">
  <Tab value="architecture-diagram" label="🏗️ Layer Architecture">
    **3-Layer Architecture:**
    ```
    🖥️  PRESENTATION LAYER
    ├── React Server Components (RSC)
    ├── Client Components ("use client")
    ├── Server Actions (mutations)
    └── DAL (Data Access Layer with cache)
                    ⬇️
    ⚙️  SERVICE LAYER
    ├── Business Logic
    ├── Validation (Zod schemas)
    ├── Authorization (CASL RBAC)
    ├── Facades (interfaces)
    └── Interceptors (logging, audit)
                    ⬇️
    🗄️  PERSISTENCE LAYER
    ├── Drizzle ORM
    ├── Database Models
    ├── Repositories (DAO pattern)
    └── Type Transformations
    ```
  </Tab>

  <Tab value="data-flow" label="🔄 Data Flow">
    **Read Operations (Queries):**
    ```
    Component → DAL (with cache) → Facade → Service → Repository → Database
    ```

    **Write Operations (Mutations):**
    ```
    Server Action → Facade → Service (validation + auth) → Repository → Database
    ```

    **Key Benefits:**
    - Cached reads with `react-cache`
    - Validated writes with business rules
    - Authorized access at service level
    - Clean separation of concerns
  </Tab>

  <Tab value="layer-rules" label="📋 Layer Rules">
    **Strict Layer Communication:**

    - **Presentation** → Only calls DAL (reads) or Server Actions (writes)
    - **DAL** → Transforms DB models to domain types
    - **Service** → Contains business logic + validation + authorization
    - **Repository** → Handles database operations only
    - **No layer bypassing** - Each layer can only call the next one

    **Type Separation:**
    - Presentation uses **Domain Types**
    - Services use **Domain Types**
    - Persistence uses **Drizzle Types**
  </Tab>
</Tabs>

## Example: Complete Data Flow

Here's how a typical user operation flows through all layers:

<Tabs defaultValue="read-example">
  <Tab value="read-example" label="📖 Read Operation">
    **Get User Profile:**
    ```typescript
    // 🖥️ PRESENTATION (RSC Component)
    import { getUserByIdDal } from '@/app/dal/user-dal'

    export default async function UserProfile({ userId }: { userId: string }) {
      const user = await getUserByIdDal(userId) // Cached read
      return <div>{user.name}</div>
    }

    // 📊 DAL (Data Access Layer)
    import { cache } from 'react'
    import { getUserByIdService } from '@/services/facades/user-service-facade'

    export const getUserByIdDal = cache(async (id: string) => {
      const userModel = await getUserByIdService(id)
      return transformToUserDTO(userModel) // Transform to domain type
    })

    // ⚙️ SERVICE (Business Logic)
    export const getUserByIdService = async (id: string) => {
      const parsed = userUuidSchema.safeParse(id) // Validation
      if (!parsed.success) throw new ValidationError()

      const granted = await canReadUser(parsed.data) // Authorization
      if (!granted) throw new AuthorizationError()

      return await getUserByIdDao(parsed.data) // Call repository
    }

    // 🗄️ REPOSITORY (Data Access)
    export async function getUserByIdDao(id: string): Promise<UserModel> {
      return await db.query.users.findFirst({
        where: eq(users.id, id)
      })
    }
    ```
  </Tab>

  <Tab value="write-example" label="✏️ Write Operation">
    **Update User Profile:**
    ```typescript
    // 🖥️ PRESENTATION (Server Action)
    import { updateUserService } from '@/services/facades/user-service-facade'

    export async function updateUserAction(formData: FormData) {
      "use server"

      const result = await updateUserService({
        id: formData.get('id') as string,
        name: formData.get('name') as string,
        email: formData.get('email') as string
      })

      revalidatePath('/account')
      return result
    }

    // ⚙️ SERVICE (Business Logic)
    export const updateUserService = async (data: UpdateUserInput) => {
      // 1. Validation
      const parsed = updateUserSchema.safeParse(data)
      if (!parsed.success) throw new ValidationError()

      // 2. Authorization
      const granted = await canUpdateUser(parsed.data.id)
      if (!granted) throw new AuthorizationError()

      // 3. Business Logic
      const updatedUser = await updateUserDao(parsed.data)

      // 4. Side Effects (optional)
      await sendUserUpdatedEmail(updatedUser)

      return updatedUser
    }

    // 🗄️ REPOSITORY (Data Access)
    export async function updateUserDao(data: UpdateUserData) {
      return await db.update(users)
        .set({
          name: data.name,
          email: data.email,
          updatedAt: new Date()
        })
        .where(eq(users.id, data.id))
        .returning()
    }
    ```
  </Tab>
</Tabs>
