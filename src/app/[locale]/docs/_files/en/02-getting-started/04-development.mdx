## Available Scripts

Master all the development commands organized by category for efficient development workflow.

<Tabs defaultValue="dev">
  <Tab value="dev" label="Development">
    ```bash
    # Development server
    pnpm dev          # Start development server with Turbopack
    pnpm build        # Build for production with Turbopack
    pnpm start        # Start production server

    # Code quality
    pnpm lint         # Run ESLint for code linting
    pnpm lint:fix     # Fix ESLint errors automatically
    pnpm format       # Check code formatting with Prettier
    pnpm format:fix   # Fix code formatting with Prettier

    # Environment setup
    pnpm init:env     # Interactive environment configuration
    ```
  </Tab>

  <Tab value="db" label="Database">
    ```bash
    # Database management
    pnpm db:studio    # Open Drizzle Studio (database GUI)
    pnpm db:push      # Push schema changes to database (--force)
    pnpm db:generate  # Generate migrations from schema
    pnpm db:migrate   # Run pending migrations
    pnpm db:pull      # Pull schema from database
    pnpm db:drop      # Drop database tables

    # Data management
    pnpm db:seed      # Seed database with demo data
    pnpm db:clear     # Clear all data from database
    pnpm db:reset-seed # Clear, push schema, and seed database

    # Connection testing
    pnpm db:check     # Test database connection
    ```
  </Tab>

  <Tab value="test" label="Testing">
    ```bash
    # Unit testing with Vitest
    pnpm test         # Run unit tests with Vitest (--pool=forks)

    # End-to-End testing with Playwright
    pnpm test:e2e     # Run E2E tests with Playwright
    pnpm test:e2e:ui  # Run E2E tests with Playwright UI
    pnpm test:e2e:headed # Run E2E tests with visible browser
    ```
  </Tab>

  <Tab value="stripe" label="Stripe & Tools">
    ```bash
    # Stripe development
    pnpm stripe:listen # Listen for webhooks (forward to localhost:3000)
    pnpm stripe:logs  # View Stripe logs in real-time

    # Code analysis
    pnpm knip         # Find unused dependencies and exports

    # Git hooks
    pnpm prepare      # Setup Husky git hooks
    pnpm lint-staged  # Run linting on staged files
    ```
  </Tab>
</Tabs>

## Development Workflow

### **Daily Development Flow**

<Steps>
<Step title="Start Development">
  ```bash
  # Start the development server
  pnpm dev

  # Open database GUI (optional)
  pnpm db:studio
  ```
</Step>

<Step title="Make Changes">
  - Edit code with full TypeScript support
  - Hot reload automatically applies changes
  - Database changes auto-sync with schema
</Step>

<Step title="Test Changes">
  ```bash
  # Run relevant tests
  pnpm test [pattern]

  # Run E2E tests for user flows
  pnpm test:e2e
  ```
</Step>

<Step title="Quality Check">
  ```bash
  # Check code quality before commit
  pnpm lint
  pnpm type-check
  pnpm format
  ```
</Step>
</Steps>

### **Feature Development Process**

<Steps>
<Step title="Database Changes">
  ```bash
  # 1. Update schema in src/db/models/
  # 2. Generate migration
  pnpm db:generate

  # 3. Apply to development database
  pnpm db:migrate
  ```
</Step>

<Step title="Service Layer">
  ```bash
  # 1. Add service functions in packages/services/
  # 2. Write unit tests
  pnpm test services

  # 3. Update facades if needed
  ```
</Step>

<Step title="UI Implementation">
  ```bash
  # 1. Create/update components
  # 2. Add to pages with proper data access layer
  # 3. Test UI components
  pnpm test components
  ```
</Step>

<Step title="Integration Testing">
  ```bash
  # 1. Write E2E tests for complete user flows
  # 2. Test with real data and services
  pnpm test:e2e
  ```
</Step>
</Steps>

### **Debugging Strategies**

**Database Issues:**
```bash
# Check database connection
pnpm db:check

# View data visually
pnpm db:studio

# Reset data if corrupted
pnpm db:reset-seed
```

**Build Issues:**
```bash
# Clean build artifacts
pnpm clean

# Rebuild packages
pnpm build:packages

# Check TypeScript errors
pnpm type-check
```

**Test Failures:**
```bash
# Run specific test file
pnpm test path/to/file.test.ts

# Run with detailed output
pnpm test --reporter=verbose

# Generate coverage to find untested code
pnpm test:coverage
```

## Best Practices

### **Code Organization**

<Callout type="success">
  **Layered Architecture**: Follow the 5-layer pattern for consistent, maintainable code structure.
</Callout>

1. **Presentation Layer** - React components and pages
2. **DAL (Data Access Layer)** - Cached data access with React
3. **Facade Layer** - Interface between presentation and business logic
4. **Service Layer** - Business logic, validation, and authorization
5. **Persistence Layer** - Database models and repositories

### **Development Habits**

**Before Each Commit:**
```bash
# Run the quality trio
pnpm lint          # Code quality
pnpm type-check    # Type safety
pnpm test          # Functionality
```

**Database Best Practices:**
- Always generate migrations for schema changes
- Test migrations on copy of production data
- Use transactions for multi-step operations
- Keep database seeds up to date

**Testing Strategy:**
- Write tests for business logic first
- Add integration tests for complex flows
- Use E2E tests for critical user journeys
- Mock external services in unit tests

<Callout type="info">
  **Performance Tip**: Use `pnpm dev` with Turbopack for the fastest development experience. Hot reload is optimized for large codebases.
</Callout>
